# Smart Alarm Database Setup\n\nThis directory contains the database schema and setup scripts for the Smart Alarm application using Supabase.\n\n## Prerequisites\n\n1. Create a Supabase project at [supabase.com](https://supabase.com)\n2. Get your project URL and anon key from the Supabase dashboard\n\n## Setup Instructions\n\n### 1. Configure Environment Variables\n\nCopy `.env.example` to `.env` and fill in your Supabase credentials:\n\n```bash\ncp .env.example .env\n```\n\nUpdate the values in `.env`:\n```\nVITE_SUPABASE_URL=https://your-project-id.supabase.co\nVITE_SUPABASE_ANON_KEY=your-anon-key\nSUPABASE_SERVICE_ROLE_KEY=your-service-role-key\n```\n\n### 2. Run Database Schema\n\n1. Go to your Supabase project dashboard\n2. Navigate to the SQL Editor\n3. Copy and paste the contents of `schema.sql`\n4. Run the SQL script\n\nThis will create:\n- All necessary tables (users, alarms, alarm_events, voices, user_voices)\n- Indexes for performance\n- Row Level Security (RLS) policies\n- Triggers for automatic timestamp updates\n- Default voice configurations\n\n### 3. Enable Realtime (Optional)\n\nFor real-time alarm synchronization across devices:\n\n1. Go to Database > Replication in your Supabase dashboard\n2. Enable realtime for the `alarms` table\n3. This allows the app to sync alarm changes instantly\n\n## Database Schema Overview\n\n### Tables\n\n#### `users`\n- User profiles and preferences\n- Linked to Supabase Auth\n- Stores theme, notification, and voice preferences\n\n#### `alarms`\n- User-created alarms\n- Stores time, days, voice mood, and status\n- Links to user via `user_id`\n\n#### `alarm_events`\n- Log of alarm triggers and interactions\n- Tracks dismissals, snoozes, and methods used\n- Used for analytics and debugging\n\n#### `voices`\n- Available TTS voices and configurations\n- Public table with default voice settings\n- Extensible for future TTS provider integration\n\n#### `user_voices`\n- User-specific voice customizations\n- Custom messages per voice mood\n- Links users to preferred voices\n\n### Security\n\n- Row Level Security (RLS) enabled on all user data tables\n- Users can only access their own data\n- Automatic user profile creation on signup\n- Secure API access through Supabase Auth\n\n### Performance\n\n- Indexes on frequently queried columns\n- Optimized for alarm lookup and event logging\n- View for aggregated user statistics\n\n## API Usage\n\nThe app uses the `SupabaseService` class in `src/services/supabase.ts` to interact with the database:\n\n```typescript\n// Sign up new user\nconst { user, error } = await SupabaseService.signUp(email, password, name);\n\n// Load user alarms\nconst { alarms, error } = await SupabaseService.loadUserAlarms(userId);\n\n// Save alarm\nconst { error } = await SupabaseService.saveAlarm(alarm);\n\n// Subscribe to real-time updates\nconst unsubscribe = SupabaseService.subscribeToUserAlarms(userId, (alarms) => {\n  // Handle alarm updates\n});\n```\n\n## Local Development\n\nFor local development without Supabase:\n- The app will fall back to local storage\n- User authentication will be bypassed\n- All alarm data is stored locally\n- Real-time sync is disabled\n\n## Production Deployment\n\n1. Set up environment variables in your hosting platform\n2. Configure Supabase Auth providers if needed\n3. Set up email templates for user onboarding\n4. Configure webhook endpoints for alarm notifications\n5. Set up monitoring and alerts for database performance\n\n## Backup and Recovery\n\nSupabase provides automatic backups, but for critical deployments:\n1. Set up daily database dumps\n2. Store critical alarm data redundantly\n3. Implement data export functionality for users\n4. Test recovery procedures regularly\n\n## Troubleshooting\n\n### Common Issues\n\n1. **RLS Policy Errors**: Check that user is authenticated and policies allow the operation\n2. **Connection Errors**: Verify environment variables and network connectivity\n3. **Migration Errors**: Ensure schema.sql is run completely without errors\n4. **Performance Issues**: Check query performance and add indexes if needed\n\n### Debugging\n\n- Enable Supabase logging in dashboard\n- Use browser dev tools to inspect network requests\n- Check console for error messages\n- Verify user authentication status\n\n## Future Enhancements\n\n- Integration with external TTS providers (ElevenLabs, Google Cloud TTS)\n- Advanced analytics and reporting\n- Backup and sync to cloud storage\n- Multi-device alarm coordination\n- Social features and alarm sharing
